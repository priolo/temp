STORE​

The STATE is no longer inside the React COMPONENTs but is in the STORE
So the components that the Employees serve are completely independent.

  • They have no dependencies on the parent containing them
  • You don't have to pass the params to them
  • They can be moved anywhere on the interface with a copy & paste
  • The component is clean and tidy, it ONLY takes care of the VIEW without having complex code

code

MSW​

STORES ALSO take care of making API calls.

I used MSW to simulate REST API calls
Knowing the MWS specifications allows you to start the FE with MOCK data without having the BE.

Advantages of MOCKs:

  • You can focus only on the FE without having to install the BE locally
  • You are forced to properly formalize the API (this is also a great documentation)
  • Simplify e2e testing enormously.
    For example Cypress does not have to start additional services but only run the FE statically.
