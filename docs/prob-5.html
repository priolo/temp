but since I can't use external libraries (and to do something new)

CONS​

They are not recommended for frequent updates (they render ALL their children with each change)
therefore not very performing if they have a large number of components inside

PRO​

They are simple to use and in some ways "native"
that is, available directly in React
Performances are only about updates
so if there are few updates it shouldn't cause any problems

NOTE: The CONTEXT is also used for the management of the "themes" in MUI.

PreferenceProvider (Provider)​

Create the CONTEXT and the PROVIDER that contains the REDUCE
In this way the REDUCE is available within the children of the PROVIDER
and above all it is reactive (i.e. if modified it updates the VIEW)
code

usePreferences (Hook)​

In this example the REDUCE carries out only the change of the STATE.
The HOOK is used within the VIEW
to also call the REST API.
code

NOTE: the HOOK can be called independently in any COMPONENT within the PROVIDER

VIEW​

It is very simple to use:
In my example we have two distinct and independent COMPONENTs using the HOOK

  • Command.jsx
  • Detail.jsx

when you want to retrieve / modify the PREFERENCES just use 

const [preferences, setPreference] = usePreferences()
...
const handleChange = e => setPreference("preference_name", e.target.value)
...
return <div>{preferences["preference_name"]}</div>
