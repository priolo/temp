Note

The voucher assignment form is very interesting because:

  • It offers inline editing, which is faster and more immediate.
  • Use the filter to define the block being edited
    instead of the classic checkboxes. This also simplifies a lot.

In this case I would still use the STOREs. I would do a STORE-EMPLOYEES like that (pseudo-code) 

    state: {
        // all EMPLOYEEs visible in page
        displayed: [],
        // the currently selected employee 
        employeeIdSel: null,
        // pagination data
        pagination: {
            current: 0,
            limit: 15,
            total: null,
        },
        // filter data
        filter: {
            text: "",
            groupId: null,
            officeId: null
        },
        // edit. you could make a separate STORE that deals only with modified records
        edit: {
            // STATE of the gif customisation dialogue
            isDialogTundrGiftOpen: false,
            modified: [
                { 
                    employeeId: int, total: int, date: Date, ...,
                    tundrGift: { shopping: true, ... }
                },
                ...
            ]
        }
    },

    getters: {

        // returns the properties of an EMPLOYEE
        // With these I can see the changed data in the table
        getProps: ( employeeId ) => state.edit.modified.find( e => e.id == employeeId ),

        // returns the summary
        getSummary: () => state.edit.modified.reduce ( (acc, e) => ..., {} )
    },

    action: {

        // Take from the BE all employees to be displayed
        fetch: () => {
            state.all = ajax.getAll(
                state.filter,
                state.pagination
            )
        },

        // adds a specific change 
        addModify: ( employeeId, props ... ) => {
            resetModify( employeeId )
            state.edit.modified.push ( { employeeId, ...} )
        },

        // adds bulk changes
        addModifyBulk: ( props ... ) => {
            state.edit.modified.concat([...])
        },

        // resets an modification
        resetModify: ( employeeId ) => state.edit.modified.filter( m => m.id != employeeId ),

        // reset all employees displayed
        resetModifyBulk: () => state.edit.modified.filter( m => !state.displayed.some( e => e.id == m.id ) ),

        // send everything to the BE
        save: () => ajax.post( state.edit.modified ),

        ...
    },
  • Each modification adds/updates in the array state.edit.modified status
  • In the display table the editable properties must be obtained with the "getter" getProps
  • The addition of the change could also be called only on "blur" event
  • Once the state.edit.modified is handled correctly it is trivial to send them to the server, reset them, get statistics
